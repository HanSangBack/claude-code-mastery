# 프로젝트 개요
Next.js(App Router) + TypeScript 기반 쇼핑몰 관리 앱

# 코드 스타일
- 들여쓰기: 스페이스 2칸
- 작은따옴표('') 사용

# GIT 규칙
- 커밋 메시지는 한글로 작성
- 브랜치명: feature/기능명, fix /버그명
- 커밋은 작은 단위로 나눠서

# 작업 방식
- 파일 수정 전 변경 계획을 먼저 설명
- 한 번에 너무 많은 파일을 수정하지 말 것

# 코드 컨벤션
- 컴포넌트: PascalCase  
  예: `ProductTable.tsx`, `MarginCalculator.tsx`
- 훅: use로 시작  
  예: `useProducts.ts`, `useMarginCalculator.ts`
- 스토어: camelCase  
  예: `productStore.ts`, `marginStore.ts`
- 타입: 도메인 기준 분리  
  예: `product.ts`, `sourcing.ts`, `cost.ts`
